### 二、 核心实现步骤 (Step-by-Step)

#### 第一阶段：数据清洗与 ETL (最容易失分的地方)
论坛中明确提到数据有错（比如同一机场指向多个国家），你需要做以下处理：
1. **排查并清洗脏数据**：
   * 提取所有出发和到达机场的独立列表。
   * 检查同一个机场是否有不同的 `Country` 或 `City` 记录。
   * **实操**：比如发现 *Sydney Kingsford Smith International Airport* 被标记为 Canada，你需要写 Python 脚本或手动在源数据中将其统一修正为 *Australia*。
   * **一定要把这一步截图并写在 PDF 报告中！**（“我通过查找发现...并进行了修正...”）。
2. **将大 CSV 拆分为实体表和关系表**：
   * 生成 `airports.csv`：包含 `Airport_ID`, `Name`, `City`, `Country` 等，**必须去重**。
   * 生成 `airlines.csv`：包含 `Airline_ID`, `Name`, `Alias`, `Country` 等，**必须去重**。
   * 生成 `routes.csv`：包含 `Source_Airport_ID`, `Dest_Airport_ID`, `Airline_ID`, `Codeshare`, `Equipment` (机型，注意这是分号分隔的字符串)。每行代表一个服务，**不要轻易去重**（根据Q3的要求，一行就是一个服务记录）。

#### 第二阶段：Arrows.app 图模型设计
根据指导老师的要求，画一个**Schema-level**的图：
* **节点 (Nodes)**:
  * `(Airport)` 属性：`id`, `name`, `city`, `country` 等。
  * `(Airline)` 属性：`id`, `name`, `country` 等。
* **关系 (Relationships)**:
  * 题目要求Q5为“最多3个ROUTE跳数”，所以机场之间**必须**有 `[:ROUTE]` 关系。
  * `(Airport)-[:ROUTE]->(Airport)`，关系的属性应包含运营该航线的 `airline_id` 和机型 `equipment`。
  * *(可选拓展)* 航司和航线节点的关系：可以设计 `(Airline)-[:OPERATES]->(Route)` 或者直接把航司作为属性放在 `[:ROUTE]` 里。考虑到 Q1-Q6 的查询，最简练的模型是 `(Airport)-[r:ROUTE {airline_name: '...', equipment: '...'}]->(Airport)`。

#### 第三阶段：Neo4j 导入数据 (Cypher)
1. **创建唯一性约束（提升百倍查询速度的关键）**：
   ```cypher
   CREATE CONSTRAINT FOR (a:Airport) REQUIRE a.id IS UNIQUE;
   CREATE CONSTRAINT FOR (al:Airline) REQUIRE al.id IS UNIQUE;
   ```
2. **导入 CSV**：
   使用 `LOAD CSV WITH HEADERS FROM 'file:///airports.csv' AS row` 分别创建 `Airport` 节点、`Airline` 节点，最后根据 `routes.csv` 用 `MATCH` 找到起点和终点机场，使用 `CREATE` 或 `MERGE` 建立 `[:ROUTE]` 关系。

#### 第四阶段：高难度 Cypher 查询攻克 (Q1 - Q6 核心思路)
* **Q1 (特定国家的航司)**: 基础 `MATCH` 查询加 `WHERE` 过滤。
* **Q2 (国内/国际航班统计)**: 比较起点和终点的 `country` 属性。
* **Q3 (最多记录的机场对 - 无向)**:
  * **难点**：把 A->B 和 B->A 算作同一个对。
  * **破局解法**：在 MATCH 后强行对机场排序组合。
    ```cypher
    MATCH (a:Airport)-[r:ROUTE]->(b:Airport)
    WITH CASE WHEN a.name < b.name THEN a.name ELSE b.name END AS airport1,
         CASE WHEN a.name < b.name THEN b.name ELSE a.name END AS airport2, r
    RETURN airport1, airport2, count(r) AS total_records
    ORDER BY total_records DESC LIMIT 1
    ```
* **Q4 (最多机型的机场对)**:
  * **难点**：机型是 `737;320` 这种分号隔开的字符串。
  * **破局解法**：使用 Cypher 内置函数 `split()` 拆分，然后 `UNWIND` 展开列表，最后 `COUNT(DISTINCT equipment_type)`。机场对的处理逻辑同 Q3。
* **Q5 (北京到珀斯，最多3跳)**:
  * **难点**：变长路径查询。
  * **破局解法**：`MATCH path = (start:Airport {name: "..."})-[:ROUTE*1..3]->(end:Airport {name: "..."})`。然后提取沿途节点列表以确保是 distinct routes（去重）。
* **Q6 (CITS5504 专属：航司硬碰硬竞争)**:
  * **思路**：找同一对机场，同时被航司 A 和航司 B 运营。使用 `<` 操作符确保 `Airline A` 和 `Airline B` 不会重复计算（如出现 A,B 和 B,A 两次）。
* **自定义 APOC 查询**:
  * 使用 `apoc.path.spanningTree` 做广度优先搜索，或者使用 `apoc.meta.schema()` 生成数据库元数据表（这不仅满足 APOC 强制要求，还能直接无缝衔接报告最后的 Metadata 讨论部分！**极度推荐**）。
